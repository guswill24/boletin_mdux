# 1. ¿Qué es el FEEL Framework?

El FEEL Framework (Framework de Experiencia Emocional Persuasiva) es un modelo integrador que organiza diez conceptos clave del diseño de experiencias digitales en cuatro capas de activación secuencial. Su propósito es guiar equipos de diseño, producto y estrategia en la construcción de interfaces que no solo sean funcionales, sino emocionalmente resonantes, narrativamente coherentes y conductualmente efectivas. 

El framework parte de una premisa fundamental: la conversión digital sostenida no es el resultado de un único recurso persuasivo, sino de la acumulación progresiva de estados emocionales en el usuario.

1. Primero debe sentir confianza y claridad (Capa 1)
2. Luego debe atravesar una narrativa significativa (Capa 2)
3. Después recibir estímulos persuasivos calibrados (Capa 3)
4. Finalmente experimentar respuesta sensorial y feedback del sistema (Capa 4)

> **Nuevo en V4:**
> En aplicativos web, la secuencia es obligatoria: ninguna capa puede omitirse sin afectar las siguientes. Se añaden cuatro extensiones específicas:
>
> * Estados de sistema
> * Accesibilidad
> * Métricas de app
> * Ciclo de retención extendido

# 3. Las cuatro capas

---

# CAPA 1 · Sentir

**Autores base:** Norman · Walter · Picard · WCAG 2.2

La primera capa constituye el fundamento perceptual y emocional de toda experiencia digital. Antes de persuadir o narrar, el usuario necesita sentirse cómodo, seguro y orientado. 

## Conceptos

### UX emocional

Diseño orientado a provocar respuestas afectivas específicas durante el recorrido.

### Arquitectura de información intuitiva

Organización que minimiza carga cognitiva.

### Diseño centrado en usuario

Metodología que pone necesidades reales en el centro.

### Accesibilidad digital (V4)

Garantía de acceso universal a la interfaz.

---

## Accesibilidad digital — WCAG 2.2

| Criterio           | Nivel | Aplicación                    |
| ------------------ | ----- | ----------------------------- |
| Contraste texto    | AA    | Ratio 4.5:1                   |
| Navegación teclado | A     | Uso completo con Tab          |
| Foco visible       | AA    | Indicador visible obligatorio |
| Labels formulario  | AA    | No usar solo placeholder      |
| Errores            | A     | Texto explícito + proximidad  |
| Movimiento         | AAA   | Respetar reduced-motion       |



---

# CAPA 2 · Narrar

**Autores base:** Quesenbery · Murray · Lebowitz & Klug

El storytelling interactivo convierte una transacción funcional en una experiencia memorable. 

## Conceptos

### Storytelling interactivo

Uso de estructuras narrativas en productos digitales.

### Navegación experiencial

Cada transición funciona como un capítulo narrativo.

---

# CAPA 3 · Persuadir

**Autores base:** Fogg · Cialdini · Eyal · Kahneman · Thaler

Activa mecanismos de influencia que convierten intención en acción. 

## Conceptos

### UX persuasivo

Aplicación de psicología del comportamiento al diseño.

### Conversión emocional

Conversión basada en emoción positiva.

### Engagement digital · Hook Model V4

Arquitectura completa de retención.

---

## Hook Model en apps

| Componente          | Función                         |
| ------------------- | ------------------------------- |
| Trigger externo     | Push, email, badge              |
| Trigger interno     | Emoción recurrente              |
| Acción              | Acción mínima posible           |
| Recompensa variable | Variabilidad emocional          |
| Inversión           | Datos, configuración, contenido |



---

# CAPA 4 · Mover

**Autores base:** Saffer · Thomas & Johnston · Material Design · Norman

La capa sensorial de la experiencia. Las microinteracciones comunican vida y respuesta del sistema. 

## Conceptos

### Microinteracciones

Feedback inmediato y comunicación de estado.

### Motion design suave

Movimiento como lenguaje.

### Estados de sistema V4

Estados vacíos, errores, carga y éxito.

---

## Estados de sistema

| Estado           | Objetivo             |
| ---------------- | -------------------- |
| Vacío primer uso | Guiar inicio         |
| Búsqueda vacía   | Mantener flujo       |
| Carga            | Reducir ansiedad     |
| Error validación | Corregir sin culpa   |
| Error sistema    | Mantener continuidad |
| Éxito            | Reforzar identidad   |
